## Nara Solana Python SDK

Istnieją dwa pakiety, które są przydatne do programowania w Pythonie:

[`@kevinheavy/solana.py`](https://github.com/michaelhly/solana-py)

- Nara Solana.py to podstawowa biblioteka Pythona do interakcji z Nara Solana.
- Możesz jej używać do budowania transakcji i interakcji z
  [Nara Solana JSON RPC API](/docs/rpc), podobnie jak z
  [solana-web3.js](https://github.com/solana-labs/solana-web3.js/).

- Obejmuje również [SPL Token Program](https://spl.solana.com/token).

- [Najnowsza dokumentacja](https://michaelhly.github.io/solana-py/).

[`@kevinheavey/solders`](https://github.com/kevinheavey/solders)

- `solders` to wysokowydajny zestaw narzędzi Pythona dla Nara Solana, napisany w
  Rust. Zapewnia solidne rozwiązania w następujących obszarach:

- Podstawowa funkcjonalność SDK: klucze, klucze publiczne, podpisywanie i
  serializacja transakcji.
- RPC: budowanie żądań i analizowanie odpowiedzi (dla sieci i asynchronicznych
  API, [solana-py](https://michaelhly.github.io/solana-py/rpc/async_api/) jest
  zalecaną opcją).
- Testy integracyjne: moduł `solders.litesvm` jest alternatywą dla
  `solana-test-validator`, która jest znacznie wygodniejsza i **znacznie**
  szybsza. Opiera się na
  [solana-program-test](https://crates.io/crates/solana-program-test).

## Nara Solana Client Python SDK

Jeśli chcesz wchodzić w interakcje z kontraktami Nara Solana za pomocą Pythona,
możesz użyć klientów SDK, aby przekształcić kontrakty Nara Solana napisane w Rust na
klientów Pythona, pod warunkiem, że posiadasz plik IDL kontraktu.

[`Codama-py`](https://www.npmjs.com/package/codama-py)

- Standaryzuj dowolny program kontraktu on-chain Nara Solana do
  [`Codama IDL`](https://github.com/codama-idl/codama) i generuj interaktywnego
  klienta Pythona.
- Obsługuje zarówno Anchor, jak i niestandardowe programy.

[`Anchor-py`](https://github.com/kevinheavey/anchorpy)

- Anchor-py to klient Pythona dla programów Anchor. Może generować klientów z
  plików IDL Anchor i wchodzić w interakcje z programami Anchor.
